# GrowwWise AI

GrowwWise AI is an OpenAI-powered fintech support copilot and analytics dashboard for a retail investing platform like Groww. It is built as a polished Streamlit MVP for an AI Intern interview demo, with RAG, OpenAI API usage, structured JSON outputs, issue classification, ticket summarization, portfolio risk explanation, financial safety guardrails, dataset-driven support analytics, and a modern product-style interface.

## Problem Statement

Retail investing platforms receive a high volume of support questions about SIP failures, payment status, KYC rejection, order execution, IPO applications, redemption timelines, and investing basics. Many users are beginners, so answers must be simple, grounded, and careful about financial advice boundaries. Internal support and product teams also need analytics to understand which issues drive volume, escalations, slow resolution, and low customer satisfaction.

## Solution

GrowwWise AI demonstrates a support copilot that can:

- Retrieve relevant support context from a mock help center using ChromaDB and embeddings.
- Generate grounded answers using the OpenAI Responses API.
- Classify support issues into category, urgency, escalation need, and next step.
- Summarize raw complaints into support-agent ticket summaries.
- Explain portfolio risk using Pandas calculations plus OpenAI-generated explanation.
- Block unsafe prompts asking for guaranteed returns or direct buy/sell recommendations.
- Analyze a realistic mock support ticket dataset with KPIs, charts, filters, and AI-generated business insights.

The app runs with real OpenAI calls when `OPENAI_API_KEY` is set. If no key is present, it uses deterministic mock fallbacks so the UI still works for a local interview demo.

## Features

- **Ask Assistant**: RAG assistant over mock Groww-style help docs.
- **Issue Classification**: Structured JSON output with category, urgency, human escalation, reason, and next step.
- **Portfolio Risk Explainer**: CSV-driven allocation analysis by asset type and risk level.
- **Support Ticket Summary**: Structured ticket summary with priority, sentiment, and suggested support action.
- **Analytics Dashboard**: Dataset-driven dashboard for support ticket trends, escalation drivers, resolution bottlenecks, and CSAT patterns.
- **Guardrails**: Safe refusal for direct stock tips, guaranteed returns, all-in F&O prompts, and risk-free profit claims.

## Analytics Dashboard

The analytics dashboard uses `data/support_tickets.csv`, a realistic mock dataset of 420 fintech support tickets generated by `scripts/generate_dataset.py`. It helps support and product teams understand:

- Ticket volume trends over time.
- Main user pain points by issue category and subcategory.
- Escalation drivers across KYC, Payments, SIP, Orders, and Account issues.
- Resolution bottlenecks by issue category.
- Customer satisfaction patterns by customer segment.
- Channel mix across Chat, Email, Phone, and In-App support.

The dashboard includes filters for date range, issue category, priority, status, channel, and customer segment. It also includes an AI insights button and natural language data Q&A. To protect privacy and keep prompts efficient, the app sends only aggregated summary statistics to OpenAI, not all raw ticket rows.

## UI/UX and Branding

GrowwWise AI is designed to feel like a real fintech support intelligence product demo, not a plain Streamlit script. The interface includes:

- Modern fintech dashboard styling with a light background, dark text, soft green accents, rounded cards, and subtle shadows.
- Custom Streamlit CSS centralized in `src/ui.py`.
- Branded page headers, sidebar navigation, and a text-based `GrowwWise AI` identity.

## Setup

1. Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Configure your API key:

```bash
cp .env.example .env
```

Then edit `.env`:

```bash
OPENAI_API_KEY=your_openai_api_key_here
```

The app still runs without a key using demo-mode mock fallbacks.

## How To Run

Generate or refresh the support ticket dataset:

```bash
python scripts/generate_dataset.py
```

Start the Streamlit app:

```bash
streamlit run app.py
```

Open the local URL shown by Streamlit, usually:

```text
http://localhost:8501
```
